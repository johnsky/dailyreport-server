0.1.6

0.1.5
 Добавления:
 - Использование наборов данных. Набор данных - это список столбцов таблицы с данными
 - Использование полей наборов данных. Поле набора данных - это один столбец таблицы
 - Использование ролей. Назначение ролей групам пользователей и отдельным пользователям. В системе 
 каждый пользователь может выполнять одну из ролей: Администратор, Менеджер, Редактор, "Тот кто просматривает"
 - Интегрированы ролевые политики с набором данных. Благодаря этому есть возможность управлять
 доступом к наборам данных на основе ролей
 - Интегрированы ролевые политики с полями наборов данных. Благодаря этому есть возможность 
 управлять доступом к отдельным полям наборов данных
 - Интегрированы ролевые политики с элементами управления приложения: окно свойств котельной, окно свойств набора данных.
 - Добавлено понятие "Категория расхода воды". Каждый расход воды указывается для определенной
 категории расхода воды. По умолчанию в системе для каждой котельной создана категория 
 расхода воды с названием "Общий расход". У одной котельной может быть несколько категорий расхода 
 воды
 - Добавлено понятие "Информация о топливе". Каждый расход, остаток и поступление топлива соответствует 
 определенной "Информации о топливе". Информация о топливе сожерит 2 поля "Наименование топлива",
 "Марка топлива". У одной котельной может быть несколько видов топлива, соответственно несколько 
 "информаций" о топливе
 - Переделана форма свойств котельной, добавлены информация о топливе и котегории воды. Интегрирована проверка роли
 - При редактировании данных обновляется/пересчитывается весь период. При редактировании расхода 
 топлива обновляются все записи расхода топлива, обновляются остатки топлива на каждый день для 
 котельной
 - Отображение набора данных за период
 - Реализовано отображение набора данных в отдельном окне, реализовано переключение между ними
 - Реализована возможность обновлять таблицу с данными без повторного открытия окна с набором данных
 - Реализован частично доступ к свойствам набора данных
 - Реализован быстрый доступ для создания категории воды и вида топлива из контекстного меню
 - Плановые значения указываются один раз в месяц первого числа:
    Плановый расход топлива, т
    Плановый ТЭ (топливный эквивалент)
    Плановый расход воды, м3
    Плановый расход электричества, кВтч
 - Исправлен шаблон сводного ответа: поля  отчета соответствуют указанным в служебных записках
 
  
 Известные проблемы:
 - Возникновение коллизий при создании записей: создается 2 записи в одно и то же время и сохраняются, 
 в результате по одной котельной на одно число появляется 2 записи.
 - Не работает сохранение данных из окна свойств набора данных
 - Не работает сводный отчет 
 - Не работает сохранение свойств котельной
